# Development Guide

## Project Structure

```text
skillpack/
├── src/                 # TypeScript source for the CLI
│   ├── cli.ts
│   ├── commands/
│   └── core/
├── runtime/             # Runtime template copied by init and bundled by build
│   ├── server/          # Runtime backend
│   ├── web/             # Runtime web UI
│   ├── start.sh         # macOS/Linux launcher
│   ├── start.bat        # Windows launcher
│   └── README.md
├── skillpack.json       # Current pack config, generated by create
└── skills/              # Installed skills
```

## Build the CLI

```bash
npm install
npm run build
```

## Test the CLI

After building from the project root, you can run:

```bash
# Test interactive creation and write output to a directory
npm run create -- output
node dist/cli.js init output --config ./fixtures/skillpack.json
```

## Debug the Runtime Bundle

The `runtime/` directory is treated as a template. `init` expands its contents into the pack root, and `build` writes the same relative paths into the zip while skipping any `node_modules/` directories.

The template can also be run independently for debugging:

```bash
cd runtime/server
npm install
# PACK_ROOT should point to a directory containing skillpack.json and skills/
PACK_ROOT=/path/to/your/pack-dir node index.js
PACK_ROOT=../../output node index.js
```

Open [http://127.0.0.1:26313](http://127.0.0.1:26313) in your browser to debug the web UI.
